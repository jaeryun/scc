# AI 행동 원칙

## 구현 전
- 가정을 명시하고, 모호하면 질문할 것
- 여러 해석이 있으면 모두 제시하고 선택하지 말 것
- 더 단순한 접근법이 있으면 제안할 것

## 구현 중
- 요청된 것만 구현, 추측성 기능 추가 금지
- 단일 사용 코드에 추상화 생성 금지 (단, 단일 API 호출로 충분한 단순 기능은 예외 — `types.ts` → `service.ts` → `queries.ts` 계층 유지)
- 불가능한 시나리오에 대한 에러 처리 금지
- 200줄을 50줄로 줄일 수 있으면 다시 작성

## 코드 수정
- 인접 코드, 주석, 포매팅 "개선" 금지
- 깨지지 않은 것 리팩터링 금지
- 기존 스타일에 맞출 것 (다르게 하고 싶어도)
- 자신의 변경으로 생긴 고아(미사용 임포트 등)만 제거
- 모든 변경은 사용자 요청으로 추적 가능해야 함

## 목표 주도 실행
- 작업을 검증 가능한 목표로 변환
- "버그 수정" → 재현 테스트 작성 → 통과 확인
- "리팩터링" → 전후 테스트 통과 확인
- 다단계 작업: 각 단계별 검증 기준 명시
