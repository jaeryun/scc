스위치 포트 매핑 UI — 어떤 레이아웃이 좋을까요?

IB Switch 예시: 연결상태, 포트정보(인덱스/샤시/모듈/포트/서브포트) ↔ 호스트정보(호스트네임, NIC, IP, GPU, Speed)

A: 컴팩트 로우 테이블

모든 포트를 단일 테이블에 압축. 좌측 포트정보, 우측 호스트정보를 한 행에 표시

장점

  • 많은 포트를 한눈에
  • 정렬/필터링 쉬움
  • 익숙한 엑셀 방식

단점

  • 넓은 화면 필요
  • 시각적 매핑감 약함

각 포트를 독립 카드로. 상단은 포트 ID, 하단은 연결된 호스트 정보

  • 각 포트의 상세정보 강조
  • 다양한 화면 크기에 반응형
  • 연결/미연결 시각적 차이 큼

단점

  • 화면 1개당 보이는 포트 수 적음
  • 필터/정렬 기능 구현 복잡

C: 아코디언 (접이식)

기본은 포트 목록만. 클릭시 연결정보 펼침. 깔끔하지만 매핑을 보려면 클릭 필요

호스트 gpu-srv-01 / ib0.8001 / 192.168.1.10 / A100 / 200Gb/s
🟢 Port 1/1/2 → gpu-srv-02 ›
포트 인덱스: 2, 샤시: 0, 모듈: 1, 포트: 1, 서브: 2
호스트 gpu-srv-02 / ib0.8001 / 192.168.1.11 / H100 / 200Gb/s
🔴 Port 2/1/1 (미연결) ›
🟢 Port 2/1/2 → storage-01 ›
포트 인덱스: 4, 샤시: 0, 모듈: 1, 포트: 2, 서브: 2
호스트 storage-01 / ib0.8001 / 192.168.1.20 / 100Gb/s
🟢 Port 2/2/1 → gpu-srv-03 ›

장점

  • 깔끔하고 공간 효율적
  • 포트 수 많을 때 적합

단점

  • 매핑을 보려면 매번 클릭
  • 전체 상황 파악 어려움
